# Architecture Decision Records (ADR)

本ディレクトリでは、ポートフォリオの設計・実装において下した技術的な意思決定を記録しています。

## ADR とは

ADR (Architecture Decision Record) は、ソフトウェア開発における重要な技術的決定とその背景・理由・トレードオフを文書化したものです。後から「なぜこの設計にしたのか」を振り返れるようにすることで、保守性と知識の継承を向上させます。

## ステータスの定義

| ステータス | 意味 |
|-----------|------|
| 提案中 | 検討段階で、まだ採用されていない |
| 採用済み | 採用され、実装に反映されている |
| 廃止 | 以前は採用されていたが、別の決定に置き換えられた |

## テンプレート

新しい ADR を追加する際は、以下の構成に従ってください。

```markdown
# ADR-XXX: タイトル

## ステータス
提案中 / 採用済み / 廃止

## コンテキスト
決定が必要になった背景・状況

## 決定
何をどうすることにしたか

## 理由
なぜその決定をしたか

## トレードオフ・注意点
決定によるデメリットや将来の懸念
```
